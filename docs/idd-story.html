The Problem

Agent development
has no shared grammar.

What frameworks abstract

  • API call patterns
  • Token management
  • Tool routing

The machinery of execution.

What they should abstract

The Five Primitives

Five questions. Orthogonal.

Primitive Question Defines Key Fields
Agent WHO does this? Identity, capabilities, constraints name, role, tools
Context WHAT do they know? Knowledge, state, references type, content, scope
Behavior HOW should they work? Patterns, protocols, standards pattern, rules
Intent WHY are we doing this? Goal, success criteria, scope goal, criteria, scope
Trigger WHEN does it happen? Conditions, sequences, timing activation, conditions

The Decomposition Test: If a component answers more than one question, decompose further.

Three Layers

Voice → Grammar → Machinery

Layer 1
Voice
Natural language. What the user says. No syntax, no schema.

Grammar is the lingua franca of multi-agent coordination — the intermediate representation
between what the user said and what the system does.

The Paradigm Shift

MVC structured how to build the solution.
IDD structures how to express the problem.

Property MVC IDD
Organizes Implementation code Problem expression
Audience Developers building systems Anyone directing agent work
Artifact Source files in a project A declaration of structured intent
Composability Limited (frameworks are rigid) Native (orthogonal primitives)
Verification Tests, type checks Human review of intent before action

The problem decomposition is the solution specification.
Amplifier says “here are Lego bricks.” IDD says “here is a language for describing what to build with them.”
